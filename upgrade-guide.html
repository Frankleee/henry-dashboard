🔄 Henry 运维指南 v1.0

Henry（Hermes Agent）和 Mike（OpenClaw）的升级维护操作手册

⚠️ 核心原则：Agent 不能自己升级自己。Gateway restart 会杀死当前进程。必须由另一个 Agent（或人工）执行。

一、交叉更新协议

通用流程（A 升级 B）

步骤HenryMike
1. 确认版本hermes --versionopenclaw --version
2. 备份cp -r ~/.hermes ~/.hermes.bak.DATEcp -r ~/.openclaw ~/.openclaw.bak.DATE
3. 拉取hermes updatenpm update -g openclaw
4. 依赖pip install -r requirements.txtnpm 自动完成
5. 重启launchctl kickstart -k gui/$(id -u)/ai.hermes.gatewaylaunchctl kickstart -k gui/$(id -u)/ai.openclaw.gateway
6. 验证等 5 秒 → 检查进程 → 发测试消息

升级前必查

  • OpenClaw：新版本是否有 channel 从 built-in 变为 external plugin？
  • Hermes：新版本是否有 provider 迁移？secret redaction 默认值是否变更？
  • Hermes：自定义 provider（百度/阿里/OpenCode Go）是否兼容？

二、Henry 升级详解

推荐方式

hermes update          # 最新稳定版
hermes update --pre    # 预发布版

手动 git 升级

cd ~/.hermes/hermes-agent
git fetch origin
git checkout <version-tag>
pip install -r requirements.txt
hermes gateway restart

升级后检查清单

  • hermes --version 确认版本号
  • hermes doctor 健康检查通过
  • hermes config list 无异常警告
  • 微信发消息收到回复
  • 子 agent delegate_task 能正常工作
  • Cron job 下次运行正常
  • 所有自定义 provider 可用

破坏性变更应对

风险缓解
Custom provider 失效提前备份 config.yaml，升级后 hermes doctor
Redaction 破坏输出升级后 hermes config set security.redact_secrets false
Cron 被误判注入检查 cron prompt 无明显注入特征
Session 不兼容备份 ~/.hermes/sessions/

回滚方案

cd ~/.hermes/hermes-agent
git checkout <旧版本tag>
pip install -r requirements.txt
hermes gateway restart

三、Mike 升级详解

npm update -g openclaw
openclaw --version
  • openclaw plugins list 确认 channel 在线
  • 飞书发消息收到回复
  • 微信 bridge 正常
  • 重启后进程稳定（等 30 秒再验证）

四、救援协议

交叉更新是救援协议的上层补充：

救援协议：A 挂了 → B 重启 A 的 gateway（不改配置）
交叉更新：A 要升级 → B 帮 A 升级 + 重启

两者共享安全原则：只碰对方 gateway，绝不改对方 config 文件。

救援入口：
Mike 救援 → Frank 通过 Henry 微信远程救援 Mike（mike-rescue skill）
Henry 救援 → Frank 通过 Mike 飞书远程救援 Henry（henry-rescue skill）
